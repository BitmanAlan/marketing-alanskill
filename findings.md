# Findings: 两地快消工作流归纳

## 操作路径梳理（实际执行顺序）

1. **数据获取**：根据任务调用 **Apify skill**（或 Apify API）与 **其他 API**（如今日热榜、项目脚本），拉取海外（TikTok/Facebook）与中国（抖音/B站）数据，产出 **海外快消趋势报告** 与 **中国快消趋势报告**。这是关键第一步，没有报告则后续无法进行。
2. **输入准备**：确认海外报告、中国报告就绪（平台、数据范围、关键词/热点）；若需 Obsidian 发布，确认 vault 路径与源报告笔记名。
3. **两地报告合并**：统一表格（平台、数据范围、条数），产出「两地核心快消词频表」与「两地最核心 5 类词」。
4. **核心词频**：从合并报告中提炼「用于市场分析」的核心词（海外 5 类 + 中国 5 类 + 跨区共性）；若与合并同步可省略单独提取步骤。
5. **可视化**：用 Mermaid 产出「两地平台×品类对比图」与「核心词频 mindmap」。
6. **市场机会解读**：依据 market-listening Skill（xhs-douyin-market-listening）Step 4～5，产出品类机会、卖点/痛点、内容模板、可执行清单。
7. **Obsidian 发布**：将合并报告与机会解读写入 vault，并与既有笔记（Apify报告、中国报告）建立双向 `[[wikilink]]`。

## 有效做法 vs 无效做法（来自实战）

| 有效做法 | 无效做法 | 原因 |
|----------|----------|------|
| 合并时统一「品类/场景」维度，做两地对照表 | 只并排两段文字不建表 | 表结构便于后续词频提取与可视化 |
| 用 Mermaid graph + mindmap 两种图（对比 + 树状） | 只做一种图 | 对比图看地域差异，mindmap 看全局结构 |
| 市场解读前先锁定「两地最核心 5 类词」 | 直接用原始热词做解读 | 核心词收敛后解读可执行、可优先级排序 |
| Obsidian 内新笔记引用源报告 + 源报告附录/正文引用新笔记 | 只单向链接或只放文件不链接 | 双向链接后反向链接面板与图谱才有价值 |

## 依赖关系

- **数据获取** → 产出海外报告 + 中国报告，是「合并」的唯一输入。
- 合并报告 → 是「可视化」与「核心词频」的输入。
- 核心词频（可内嵌在合并报告中）→ 是「市场机会解读」的输入。
- 合并报告 + 机会解读 → 是「Obsidian 发布」的输入。

## 常见错误与踩坑（实战归纳）

| 错误做法 | 后果 | 正确做法 |
|----------|------|----------|
| **先做合并、再拉数据** | 没有海外/中国报告，合并无输入 | 必须先执行数据获取，产出两份报告后再合并 |
| **只保留原始 JSON/CSV，不写 Markdown 报告** | 下游合并与解读无法直接消费，需重复整理 | 数据拉取后立即整理为「海外报告」「中国报告」两篇 Markdown，含平台、范围、关键词/热点表 |
| **合并时只并排两段文字，不做统一表** | 无法系统做词频对照与可视化 | 用统一「品类/场景」维度建两地对照表 |
| **市场解读直接用全部热词** | 结论散、无法排优先级、难以落地 | 先收敛为「两地最核心 5 类词」，再基于核心词做品类机会与可执行清单 |
| **只做词云/声量，不做「所以呢？该做什么？」** | 数据好看但不产生决策 | 每条洞察对应到动作：选品、话术、内容模板、责任人与优先级 |
| **Obsidian 只在新笔记里链源报告** | 反向链接不完整，图谱缺边 | 源报告附录或正文也要增加对合并报告、机会解读的链接，形成双向链 |
| **输出路径写死为某台机器的绝对路径** | 换电脑或换项目后 skill 不可用 | 用「当前项目根目录下的 output/」或「用户指定的输出目录」，在 README 中说明配置 |

## 可复用资产

- 合并报告结构：一总览表 + 二核心词频表（品类对照 + 两地最核心 5 类）+ 三 Mermaid 图 + 四来源与数据集。
- 市场机会解读结构：核心词摘要 → 品类机会（高/中优先级 + 娱乐借势）→ 卖点/痛点 → 内容模板 → 可执行清单 → 度量建议。
- Obsidian 链接约定：源报告在「附录/数据集索引」或「可延展方向」中增加对合并报告与解读的链接；新笔记在开头用「来源」「依据」指向源报告与彼此。
